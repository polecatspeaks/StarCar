# Fold conformance vectors - the language-neutral contract home

Status: Current

**What this directory is** (design rev 5 D18 + round-4 ruling Q8): the single,
language-neutral authority on fold semantics. Every fold implementation - the landed
pwsh detector (`scripts/Detect-Dispatches.ps1`) and the Go port the board train builds -
conforms to THESE fixtures, and the D18 cross-verifier CI job runs BOTH against them,
failing on any divergence. Vectors are declarative data; no implementation detail leaks
into them (the Q8 ruling rejected extracted-copy vectors as a Law 6 drifting duplicate).

**Vector shape** (`fold/*.json`):

```json
{
  "name": "kebab-case-id",
  "description": "what this vector pins, and which design/spec obligation it serves",
  "input": {
    "records": ["array of full record objects, exactly as they would sit in a store"],
    "vocab": { "kinds": ["..."], "outcomes": ["..."] },
    "now": "ISO-8601 Z instant passed to the fold as its clock"
  },
  "expected": {
    "faults": [], "discoveries": [], "dispatches": [], "intents": []
  }
}
```

**Runner contract** (implemented per-language in each suite, NOT here):
1. Materialise `input.records` into a temp store, one file per record at
   `<subject-sanitised>/<kind>-<index>.json`. Sanitise the subject for the directory
   name only (`:` is not a legal path character on Windows; replace with `-`) - the
   fold reads `subject` from record CONTENT, never from the path.
2. Write `input.vocab` files into a temp vocab dir; invoke the fold with the temp
   store, temp vocab, and `input.now` as the injected clock (the pwsh detector's
   `-StoreRoot`/`-VocabDir`/`-Now` parameters; the Go fold mirrors them).
3. Compare `faults`, `discoveries`, `dispatches`, `intents` DEEP-EQUAL against
   `expected`. `tier` and `generated_at` are excluded from comparison (environmental).

**Provenance rule:** every `expected` block is either OBSERVED (generated by running the
landed detector on the input, then reviewed for correctness) or DESIGN-MANDATED (the
landed detector is known-wrong for this case; the vector pins the CORRECT behaviour and
lands red-first with the fix). Each vector's `description` states which. A vector whose
expected was neither observed nor design-traced is a defect.
