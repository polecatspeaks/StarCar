# Fold conformance vectors - the language-neutral contract home

Status: Current

**What this directory is** (design rev 5 D18 + round-4 ruling Q8): the single,
language-neutral authority on fold semantics. Every fold implementation - the landed
pwsh detector (`scripts/Detect-Dispatches.ps1`) and the Go port the board train builds -
conforms to THESE fixtures, and the D18 cross-verifier CI job runs BOTH against them,
failing on any divergence. Vectors are declarative data; no implementation detail leaks
into them (the Q8 ruling rejected extracted-copy vectors as a Law 6 drifting duplicate).

**Vector shape** (`fold/*.json`):

```json
{
  "name": "kebab-case-id",
  "description": "what this vector pins, and which design/spec obligation it serves",
  "input": {
    "records": ["array of full record objects, exactly as they would sit in a store"],
    "vocab": { "kinds": ["..."], "outcomes": ["..."] },
    "defaults": { "dispatch_budget_seconds": 1800 },
    "now": "ISO-8601 Z instant passed to the fold as its clock"
  },
  "expected": {
    "faults": [], "discoveries": [], "dispatches": [], "intents": []
  }
}
```

`input.defaults` is OPTIONAL (spec Amendment 2, issue #22, C3R-1 fix cycle): when present,
its `dispatch_budget_seconds` is the shop-default budget the runner supplies to the fold for
this vector's run - a language-neutral way to pin the "apply a supplied default, render
overdue" fold SEMANTIC, which is what the shop-default budget case actually is (reversing
the plan-round-1 carve-out that misclassified it as environmental IO - see the migration
note below). A vector that omits `input.defaults` runs the fold with no default threaded,
exactly as before this field existed.

**Runner contract** (implemented per-language in each suite, NOT here):
1. Materialise `input.records` into a temp store, one file per record at
   `<subject-sanitised>/<kind>-<index>.json`. Sanitise the subject for the directory
   name only (`:` is not a legal path character on Windows; replace with `-`) - the
   fold reads `subject` from record CONTENT, never from the path.
2. Write `input.vocab` into a temp vocab dir as ONE FILE PER KEY, named `<key>.json`
   (`input.vocab.kinds` becomes `kinds.json`, `input.vocab.outcomes` becomes
   `outcomes.json`), each with the shape `{"values": [...]}` - the exact layout
   `schema/vocab/` uses and the detector reads `[SB-5, folded: two independent
   runner-authors must not have to open the detector to learn this]`. When
   `input.defaults.dispatch_budget_seconds` is present, write it to a temp file in the
   SAME shape `config/harness-defaults.json` uses (`{"dispatch_budget_seconds": N}`) and
   supply it to the fold - the pwsh detector via `-DefaultsPath` pointing at the temp
   file, the Go fold via its default-budget option/parameter (never the real config
   file, so a vector's expected behaviour never depends on that file's mutable value).
   Invoke the fold with the temp store, temp vocab, `input.now` as the injected clock,
   and the optional default (the pwsh detector's `-StoreRoot`/`-VocabDir`/`-Now`/
   `-DefaultsPath` parameters; the Go fold mirrors them).
3. Compare `faults`, `discoveries`, `dispatches`, `intents` DEEP-EQUAL against
   `expected`. `tier` and `generated_at` are excluded from comparison (environmental).
   Deep-equal on `faults` means fault STRINGS pinned by a vector are cross-language
   contract surfaces - both implementations emit them byte-identically. This is
   deliberate and disclosed (spec YB-8), scoped to vector-covered cases only. A
   dispatch entry's `budget_source` (`record` or `default`) is part of this same
   deep-equal comparison, present exactly when `budget_seconds` is non-null (spec
   Amendment 2 item (b) - kept minimal, never a phantom source for a null budget).

**Provenance rule:** every `expected` block is either OBSERVED (generated by running the
landed detector on the input, then reviewed for correctness) or DESIGN-MANDATED (the
landed detector is known-wrong for this case; the vector pins the CORRECT behaviour and
lands red-first with the fix). Each vector's `description` states which. A vector whose
expected was neither observed nor design-traced is a defect.

**Migration note (Q8 ruling, plan-rung task) - DONE (plan task 3.1, 2026-07-23; amended at
the C3R-1 fix cycle, 2026-07-23):** the `Detector.Tests.ps1` cases expressible as pure fold
semantics (records + vocab + now -> outputs) migrated into this directory as declarative
fixtures, red-first, proven by the existing Pester assertions passing green against the same
cases rehomed (`scripts/tests/Detector.Tests.ps1`'s fixture-driven runner). Three cases
stayed imperative per the plan-round-1 amendment (spec 7b.1) - unreadable-vocab-dir,
unreadable-defaults, and the `tier` assertion (a field this contract excludes from
comparison) - each named in `Detector.Tests.ps1`'s own header comment.

**Reversal (spec Amendment 2, issue #22, C3R-1):** the FOURTH case originally carved out -
"the shop-default budget case" - was a false premise. Reading `config/harness-defaults.json`
is IO; applying its value to render `overdue` is a fold SEMANTIC that changes rendered
state, proven when the Car 3 review round 1 reviewer constructed a byte-identical
budget-less dispatched record on which the landed pwsh detector rendered `overdue`/`1800`
and the (at the time un-threaded) Go fold rendered `dispatched`/`null` - a real divergence,
invisible to the D18 cross-verifier because no vector exercised the budget-less
lone-dispatched path. `input.defaults` (above) now carries this semantic language-neutrally,
`fold/budget-less-past-injected-default.json` pins it, and both fold bodies thread the
supplied default. The one imperative test that used to be this case's sole carrier is KEPT,
re-scoped to the genuinely-environmental detail that remains: does `-DefaultsPath`'s own
unset-default resolve to the real `config/harness-defaults.json` file when no override is
given.
